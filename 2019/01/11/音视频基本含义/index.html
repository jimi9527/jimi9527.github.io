title: 音视频基本含义

音视频技术

流媒体协议技术、封装技术、视频压缩编码技术、音频压缩编码技术

流媒体协议

### Http 协议
渐进式的下载流媒体本质是先下载文件，在利用文件本身的头部信息和部分视频帧数据然后解码播放，HLS(Http live Steaming)自适应码率流，根据网络的状况选择不同的码率的视频流，在服务端将数据存储成连续很短时长的媒体文件，客户端不断下载这些文件。
### Rtmp 协议
实时信息传输协议：将媒体数据封装成消息，消息在分成消息块，然后通过tcp协议发送到客户端，客户端收到消息块重新组合，然后在进行解封装恢复出媒体数据。

封装技术

### MP4 
MP4视频文件格式是基于QuickTime容器格式自定义的，媒体数据和媒体描述是分开的，MP4的所有数据都是封装在box 中，相当于一个数据块，box 也可以包含另一个box 这种被称为：container box,box 有且只有一个Filetype(ftyp)类型的box,包含一些文件的信息，有且只有一个moov 类型的box,是一种container box,包含文件媒体的信息。
### MKV
MKV是建立在EBML(类似XML格式的可扩展二进制语言)的基础上
每个EBML 元素都由ID SIZE DATA 三部分组成的。
MKV 文件分为EBML Header 和 Segment  EBML Header 包含了文件的版本和文档类型等相关信息，Segment部分保存了媒体文件的视频和音频的世纪数据。
### AVI 
几乎支持所有的音视频编码格式 不支持边下边播

音视频编码技术

### H264
1 H264原始码流由一个接一个的NALU组成的，一个NALU 包括 视频 编码的NAL头部信息+ RBSP （在原始编码数据后面填加了结尾比特）。
2 一帧图片编码成一个片或者多个片Slice, NALU就是装载这些片的载体
3 一个片包含片头和片数据，片头中包含分片的类型，分片的宏块类型，分片帧的数量，分片属于哪个图片以及对应帧的设置和参数，分片数据中则是宏块（存储像素数据的地方）
4 宏块包含包含了宏块类型、预测类型、Coded Block Pattern、Quantization Parameter、像素的亮度和色度数据集等等信息，每一个像素的亮度和色度信息，一个宏块包含一个16X16亮度像素和附加的一个8×8 Cb和一个 8×8 Cr 彩色像素块组成。
### AAC
  1AAC 音频格式有ADIF（Audio Data Interchange Format 音频数据交换格式）在明确定义的位置开始进行解码只有一个统一的头， 和 ADTS（Audio Data Transport Stream 音频数据传输流) 解码可以在任何位置每一帧都有一个头
  2 ADTS 由 ADTS_header 包含了采样率、声道数、帧长度一般都是7个字节，如果包含CRC 就9个字节，和AAC ES 组成
